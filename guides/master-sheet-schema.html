3. The 6 data layers — how data flows through the business

The Sheet currently starts at L2 (Contacts). The funnel above L2 is invisible. The funnel below L4 (Invoices, Reviews) is fragmented across Xero and the Sheet. This section names the layers explicitly so we know what's missing.

L1 — Enquiry MISSING

Every inbound signal: phone call (answered or missed), website form, GBP message, FB/IG DM, walk-in. Most enquiries don't become customers — but we still need to know they happened, when, and through which channel. Without L1, lead-response-time and missed-call rate cannot be measured.

Where it goes: NEW Enquiries tab.

L2 — Contact Schema live, 0 rows

An Enquiry that became a real customer (we have their full name + a way to reach them and they've agreed to engagement). One Contact may have many Jobs over time.

Where it goes: Existing Contacts tab (Master Sheet).

L3 — Quote Schema live, 0 rows

A priced scope sent to a Contact. Has a status lifecycle: Draft → Sent → Viewed → Accepted / Rejected / Expired. Drives the quote-to-book metric.

Where it goes: Existing Quotes tab + the Quote Google Doc template lives in 05 - Operations & SOP/templates/.

L4 — Job Schema live, 0 rows

An accepted Quote that's been scheduled and either is happening / has happened. Status: Scheduled → In Progress → Complete → Invoiced → Paid → Reviewed.

Where it goes: Existing Jobs tab + per-job photo set in 01 - Customers/[Customer]/[date-job]/.

L5 — Invoice Lives in Xero

Xero owns this layer. The Sheet's Invoices tab needs a rethink — three options:

  1. Delete the Invoices tab — Xero is the source of truth, don't duplicate.
  2. Mirror minimal columns (Invoice # · Job # · Amount · Status) via Xero webhook → Apps Script. Makes the Sheet self-contained for at-a-glance views without Xero login.
  3. Replace with a Cash Flow View tab — pulls Xero outstanding-receivables totals, shows ageing buckets. Different shape from current schema.

Recommendation: Option 2 for now (mirror) so Emma doesn't need Xero login for the daily glance. Wire via Zapier or Apps Script when we hit Sprint 4. Until then, leave the empty Invoices tab as a placeholder.

L6 — Review Schema live, 0 rows

The post-job feedback layer. SMS sent → review received → response posted. Drives GBP ranking.

Where it goes: Existing Reviews tab. Could be derived from Jobs (Review Requested? + Review Received? columns) but a dedicated tab keeps the response cycle visible.
