Calendar & Job-Naming Conventions

One shared calendar. One naming pattern for every job. Done right, the calendar becomes the single source of "what's on this week" for Beau, Emma, and Tim.

The rule: if a job isn't on the shared calendar, it didn't happen. The calendar is the source of truth for capacity, scheduling, and the Jobs tab. Quotes accepted but not in the calendar = not booked.

1. The shared calendar — what + why

Name: CWP Operations

Owned by: enquire@coastalwaysplumbing.com (the company, not a person)

Shared with: Beau (Editor), Emma (Editor), Tim (Viewer)

Visible on: Beau's iPhone Calendar app, Emma's phone + laptop, anyone signed in as enquire@

Why a separate calendar (not just enquire@'s default)?
Three reasons:
  • Subscribable — Beau's iPhone Calendar can subscribe to it without him logging into Workspace. He just sees CWP jobs alongside his personal calendar.
  • Filterable — Emma can hide it when she's off the clock, show it during office hours.
  • Owner-stable — owned by the company email, not a person. Survives team changes.

3. The job-naming convention

YYYY-MM-DD · Suburb · Customer surname · Job type

Examples:

  • 2026-05-02 · Wyong · Anderson · Hot water
  • 2026-05-03 · Tuggerah · Strata #47 · Blocked drain
  • 2026-05-05 · Terrigal · Patel · Bathroom roughin (Day 1 of 3)
  • 2026-05-07 · The Entrance · Vincent Realty · Leak detection
Why this format
  • Date first — sorts cleanly when you scroll back through history.
  • Suburb second — Beau scans for routing. Two Wyong jobs same day = chain them. Wyong + Bateau Bay split = plan around it.
  • Customer name third — instant recall on the day. Strata jobs use building name or PM name (e.g. "Strata #47" or "Vincent Realty").
  • Job type last — pulled from the same dropdown as the Master Sheet's Service Type column, so Sheet and Calendar match.

Service type — use exactly these strings (matches Master Sheet dropdown)

  • Emergency
  • Blocked drain
  • Gas fitting
  • Hot water
  • Leak detection
  • Bathroom reno
  • Other (only if none of the above fit)

7. Emma's calendar workflow — the daily rhythm

When Emma adds a new job (2 min)

  1. Quote accepted → open CWP Operations calendar
  2. Click the time slot → use the convention: YYYY-MM-DD · Suburb · Customer · Service
  3. Set duration (default 1 hour for standard, 2-3 for HWS, 4+ for renos)
  4. Add description fields (Customer / Address / Job # / Quote $ / Scope / Materials / Access / Notes)
  5. Pick colour: Blueberry confirmed, Sage if strata, Flamingo if emergency
  6. Send invite to Beau and (if external) the customer
  7. Cross-check: open Master Sheet → Jobs tab → add a row matching the calendar entry. Calendar and Sheet stay in sync.

When a job needs rescheduling

  • Drag the calendar event to the new slot — it'll prompt to notify guests, click Send
  • Update the Jobs tab → Date column on the same row
  • SMS the customer using script #11 (Job Booking Confirmation, with new time)

When a job is complete

  • Don't delete the past event — it's the audit trail
  • Edit the title to prepend [DONE] — e.g. [DONE] 2026-05-02 · Wyong · Anderson · Hot water
  • Update Jobs tab → Status = Complete · Review Requested = TRUE · timestamp
  • Send review SMS within 2 hours (script #2)

8. Beau's calendar workflow — the absolute minimum

Beau's only calendar duty
Read it. He doesn't fill it. Emma owns the writes.

If Beau wants to add something himself (e.g. blocks an evening, adds a memo), he can — but the discipline is: Emma is the calendar admin. Beau telling Emma "block out Thursday 2pm onwards, going to school sports" beats Beau opening Calendar himself. Less risk of double-bookings, less context-switching for him.

10. Common questions Emma will ask in the first month

QuestionAnswer
"Customer wants Tuesday morning, Beau already has 2 jobs that morning"Look at the suburbs — if all 3 are within 15min of each other, chain them. If not, offer afternoon or Wednesday.
"Two PMs both want Thursday"The PM with the firmer SLA wins (check Strata Directory tab → SLA Response Required). The other gets a courtesy SMS offering Friday or next week.
"Customer wants Saturday 3pm"Outside Saturday window — offer next available Mon morning, or upcharge if it's truly an emergency.
"Beau says don't book me Friday"Block the day Graphite. No questions asked.
"I can't tell if a slot is free"Switch to Week view. Empty = free. Don't trust Day view alone — it can hide subcontract blocks.
